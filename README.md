# LI.FI Quote Tracker

Tracks how **lifiIntents** quotes compare to every competing bridge/solver returned by the LI.FI routes API вЂ” across chains, pairs, and sizes. Runs locally against SQLite, or hosted on Vercel as a static site fed by a daily GitHub Actions cron.

For each route, it calls `@lifi/sdk getRoutes` once. The `lifiIntents` offer is extracted as the intent side of the comparison; all other returned offers (Across, Stargate, CCTP, Relay, Eco, Mayan, etc.) are the alternatives. Results land in SQLite; the web UI shows snapshot rankings and timeseries.

---

## What it tracks

- **Intent rank** вЂ” where the `lifiIntents` offer places by gross output among all competing offers (1 = best)
- **О” bps** вЂ” how many basis points lifiIntents trails the best offer
- **Intent quote / Best quote** вЂ” the raw output amounts being compared
- **Best tool** вЂ” which bridge returned the best alternative
- **Trend over time** вЂ” per-route timeseries after multiple daily runs

**Configured routes:**

| Pair | Sizes | Chains |
|------|-------|--------|
| USDC-USDC | 10, 100, 1,000, 10,000 | Ethereum в†” Base в†” Arbitrum |
| USDT-USDT | 10, 100, 1,000, 10,000 | Ethereum в†” Base в†” Arbitrum |
| ETH-ETH | 0.01, 0.1, 1 | Ethereum в†” Base в†” Arbitrum |
| WETH-ETH | 0.01, 0.1, 1 | Ethereum в†” Base в†” Arbitrum |

## Setup

```bash
pnpm install
```

Create `.env` in the repo root with your LI.FI API key (without it, you'll get rate limited to 100 RPM):

```
NEXT_PUBLIC_LIFI_API_KEY=your-key-here
INTEGRATOR_STRING=li.fi-solver
```

## Running the UI

The web UI reads static JSON from `web/public/data/`, generated by `pnpm snapshots` from `db/quotes.db`. This is the same path production uses.

```bash
pnpm snapshots   # regenerate JSON from db/quotes.db
pnpm web         # http://localhost:5173
```

Re-run `pnpm snapshots` whenever you collect new data вЂ” the UI won't see it otherwise.

> The Hono server in `server/` (`pnpm serve` / `pnpm dev`) is left intact for ad-hoc API queries against the DB but is no longer used by the web UI.

---

## Collecting data

```bash
# Pull all 84 routes
pnpm pull:daily

# Pull a specific route
pnpm pull:adhoc -- --pair USDC-USDC --from-chain 8453 --to-chain 42161 --size 100

# Fan out over all sizes for a pair/route
pnpm pull:adhoc -- --pair ETH-ETH --from-chain 1 --to-chain 8453

# Fan out over all chains for a pair + size
pnpm pull:adhoc -- --pair USDC-USDC --size 1000
```

Data is stored in `db/quotes.db` (SQLite). After collecting, run `pnpm snapshots` to regenerate the JSON files the UI reads.

---

## Resetting the database

```bash
# Stop anything reading the DB first (pnpm serve, etc.)
rm -f db/quotes.db db/quotes.db-shm db/quotes.db-wal
pnpm pull:daily
pnpm snapshots
```

## Hosted on Vercel + GitHub Actions

The deployed site is fully static. No backend, no DB at runtime вЂ” just JSON files served from a CDN. The pipeline:

```
GitHub Actions cron (daily 09:00 UTC)
  в””в”Ђ pnpm pull:daily          в†’ writes db/quotes.db
  в””в”Ђ pnpm snapshots           в†’ writes web/public/data/*.json
  в””в”Ђ force-push to `data`     в†’ orphan branch, single commit, no history bloat
  в””в”Ђ ping Vercel deploy hook  в†’ rebuild

Vercel build (on main pushes + deploy hook)
  в””в”Ђ scripts/vercel-build.sh
       в”њв”Ђ git fetch origin data && checkout into web/public/data/
       в””в”Ђ pnpm --filter web build в†’ dist/
```

The `data` branch holds **only** the latest JSON snapshot files at the root, force-pushed each run, so the repo never accumulates daily data.

### One-time setup

1. **Connect repo to Vercel** вЂ” import the GitHub repo as a new project. Vercel reads `vercel.json` automatically. If the build complains about missing `dist/`, check that the project's **Root Directory** is `./` (not `web/`).

2. **Add GitHub secrets/variables** (repo в†’ Settings в†’ Secrets and variables в†’ Actions):
   - Secret В· `NEXT_PUBLIC_LIFI_API_KEY` вЂ” your LI.FI API key (raises rate limit; collector falls back to anonymous if missing)
   - Variable В· `INTEGRATOR_STRING` вЂ” your registered LI.FI integrator (e.g. `li.fi-solver`). The collector defaults to `wtf-they-quoting` if unset, but LI.FI will reject unregistered integrators.
   - Secret В· `VERCEL_DEPLOY_HOOK_URL` вЂ” create in Vercel under Settings в†’ Git в†’ Deploy Hooks (branch `main`), then paste the URL here.

3. **Seed the `data` branch** вЂ” push to `main` doesn't have data on it. Trigger the workflow manually once: GitHub в†’ Actions в†’ `daily-pull` в†’ Run workflow. After it completes, hard-refresh the deployed site.

After that, the cron takes over at 09:00 UTC daily. Edit the cron expression in `.github/workflows/daily-pull.yml` if you want a different schedule.

### What runs where

| Step | Local | Hosted |
|---|---|---|
| Pull quotes from LI.FI | `pnpm pull:daily` | GitHub Actions |
| SQLite в†’ JSON | `pnpm snapshots` | GitHub Actions |
| Serve UI | `pnpm web` | Vercel CDN |
| Trigger refresh | (manual) | Cron + deploy hook |

## UI pages

**Snapshot** (`/`) вЂ” one row per (pair, route, size) for the selected run. Columns: Pair В· Route В· Size В· Rank В· О” bps В· Intent quote В· Best quote В· Best tool В· Alts. Filter by pair/route/size. Click *chart в†’* to open the timeseries.

**Timeseries** (`/route`) вЂ” two charts for a single route: output amount (intent vs best alt) and intent rank over time. Use the size selector to switch sweeps.

**Help** (`/help`) вЂ” step-by-step usage guide.

---

## Status semantics

- `ok` вЂ” lifiIntents competed against alternatives on this route
- `partial` вЂ” `getRoutes` returned alternatives but no `lifiIntents` offer (common for USDT-USDT and most ETH-ETH routes). Best alt is still stored and displayed.
- `error` вЂ” `getRoutes` returned nothing, or token resolution failed

## Configuration

Edit `config.json` to change the route matrix, rate limit, or alternatives cap:

```jsonc
{
  "chains": [1, 8453, 42161],
  "assetPairs": [...],
  "rateLimitRps": 1,          // API calls per second
  "alternativesTopN": 20,     // max non-lifiIntents offers stored per request
  "quoteTimeoutMs": 30000
}
```
